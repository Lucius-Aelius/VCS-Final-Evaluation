# VCS-Final-Evaluation
The main problem we are addressing is that developers often create small projects without using proper version control.
This makes it difficult to track changes and manage different versions of the project.

As a result, important updates may be lost or overwritten, and the development process becomes unorganized.

There is a need for a structured approach where changes can be tracked and managed efficiently.

Our project demonstrates how Git and GitHub can be used to solve this problem by maintaining a clear version history.
